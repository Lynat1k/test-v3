# docs/04_TIERS.md — Тарифы и типы пользователей

> Читать перед фазами 7, 8, 9, 10, 11. Источник правды по ограничениям.
> Лимиты проверяет БЭКЕНД (нельзя обходить с фронта). Хранение: SQLite.

---

## 1. Типы пользователей

| Роль | Доступ |
|---|---|
| Guest (не авторизован) | Базовый просмотр, история 1 неделя |
| Free | Авторизован, история 6 месяцев |
| Pro | Платный, расширенные лимиты |
| VIP | Платный, максимум возможностей |
| Admin | Только владелец. Доступ к админке |

> Роль хранится в SQLite (users.role). Admin задаётся вручную, не покупается.

---

## 2. Глубина истории (КРИТИЧНО — проверка на бэке)

| Роль | Доступная история графика |
|---|---|
| Guest | 1 неделя |
| Free | 6 месяцев |
| Pro | больше 6 мес (значение настраивается в админке, напр. 1 год) |
| VIP | вся доступная история из БД (futures 1 год / spot 3 года) |
| Admin | вся |

> При запросе свечей за пределами лимита роли — бэкенд обрезает диапазон и
> возвращает флаг "history_limited" (фронт показывает приглашение апгрейда).

---

## 3. Матрица возможностей

| Возможность | Free | Pro | VIP | Admin |
|---|---|---|---|---|
| Кол-во свечей графика | ограничено | больше | максимум | макс |
| Кол-во индикаторов | ограничено | больше | все | все |
| Рабочих графиков (workspace) | 1 | 2 | 2 | 2 |
| Уровни сжатия | одно (дефолт) | все 10 | все 10 | все 10 |
| Cluster Search | пресет по умолчанию, БЕЗ изменений | свои настройки + сохранение пресетов | свои настройки + пресеты | полный |
| Изменение дефолтного пресета CS | нет | нет | да | да |
| TG-уведомления (фильтрации) | нет | нет | да | да |
| Сохранение объектов рисования | нет | да | да | да |
| Доступ к админке | нет | нет | нет | да |

> Конкретные числа (свечи, кол-во индикаторов, история Pro) НЕ хардкодить —
> хранить в конфиге лимитов (SQLite таблица tier_limits), редактируемом в админке.

---

## 4. Таблица лимитов (SQLite, редактируется в админке)

```sql
CREATE TABLE IF NOT EXISTS tier_limits (
  role            TEXT PRIMARY KEY,   -- 'guest','free','pro','vip'
  history_days    INTEGER,            -- 7 / 180 / 365 / -1(=вся)
  max_candles     INTEGER,            -- лимит свечей в окне
  max_indicators  INTEGER,
  max_workspaces  INTEGER,            -- 1 / 2 / 2
  compression_all INTEGER,            -- 0=только дефолт, 1=все уровни
  cs_editable     INTEGER,            -- 0/1 редактировать Cluster Search
  cs_save_presets INTEGER,            -- 0/1 сохранять свои пресеты
  cs_edit_default INTEGER,            -- 0/1 менять дефолтный пресет
  tg_notify       INTEGER,            -- 0/1
  save_drawings   INTEGER,            -- 0/1
  updated_at      TEXT
);
```

> Значения по умолчанию для каждой роли админ задаёт через админку.
> Бэкенд читает их из этой таблицы при каждом запросе/действии.

## 5. Подписки (статус и сроки)

```sql
CREATE TABLE IF NOT EXISTS subscriptions (
  user_id      INTEGER PRIMARY KEY,
  tier         TEXT,                  -- 'free','pro','vip'
  paid_at      TEXT,                  -- дата оплаты
  expires_at   TEXT,                  -- дата истечения
  status       TEXT,                  -- 'active','paused','expired'
  updated_at   TEXT
);
```

В профиле показывать: уровень подписки, дата оплаты, дата истечения,
осталось дней, статус (active / paused / expired).

> Логика оплаты (платёжный провайдер) — отдельная задача, не на старте.
> На старте: активацию тарифа может ставить Admin вручную (заглушка под будущий биллинг).

---

## 6. Описание тарифов (для профиля и лендинга)

**Free** — ограничение по свечам, индикаторам; 1 график; одно сжатие;
Cluster Search с пресетом по умолчанию без права изменений.

**Pro** — меньше ограничений по истории; больше индикаторов; 2 графика;
все уровни сжатия; Cluster Search с своими настройками и сохранением пресетов.

**VIP** — вся история; все индикаторы; 2 графика; все уровни сжатия;
Cluster Search со своими настройками и пресетами; TG-уведомления о крупных фильтрациях.

> В конце профиля — блок «Доступные планы PROCLUSTER»: 3 тарифа со сравнением
> и кнопкой «Активировать выбранный тариф».

---

## 7. Где проверять лимиты

| Лимит | Где проверять |
|---|---|
| История (history_days) | Бэк, при запросе свечей — обрезка диапазона |
| max_candles / max_indicators | Бэк (валидация) + фронт (UX-подсказка) |
| Уровни сжатия | Бэк отдаёт только разрешённые уровни |
| Cluster Search редактирование/пресеты | Бэк (отклонять запись от Free; от Pro — свои пресеты ок) |
| Изменение дефолтного пресета | Бэк (только VIP/Admin) |
| TG-уведомления | Бэк (только VIP/Admin) |
| Объекты рисования | Бэк (сохранять только если save_drawings=1) |
| Админка | Бэк (только role=admin) на ВСЕХ admin-эндпоинтах |

> Принцип: фронт скрывает недоступное для UX, но РЕШЕНИЕ всегда за бэком.
> Никогда не доверять роли/лимиту, присланному с фронта.

## 8. Чек-лист
- [ ] Роли guest/free/pro/vip/admin в SQLite
- [ ] tier_limits с дефолтами, редактируется в админке
- [ ] История обрезается на бэке по роли + флаг history_limited
- [ ] Сжатие: бэк отдаёт только разрешённые уровни
- [ ] Cluster Search: права (Free read-only, Pro пресеты, VIP/Admin дефолт)
- [ ] TG только VIP/Admin
- [ ] Объекты рисования сохраняются по save_drawings
- [ ] Все admin-эндпоинты закрыты проверкой role=admin
- [ ] subscriptions: статус/даты/осталось дней в профиле
- [ ] Активация тарифа Admin'ом вручную (заглушка под биллинг)
