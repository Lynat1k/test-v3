# docs/03_CLUSTER_SEARCH.md — Индикатор Cluster Search

> Читать перед фазой 9. Логика как в ATAS. Базовый индикатор, включён по умолчанию для ВСЕХ.
> Пресет по умолчанию редактируют только VIP/Admin. Пресет кэшируется (Redis).

---

## 1. Суть

Ищет в свече ценовые уровни (ячейки кластера), где объём проходит фильтр.
Найденное помечается на Canvas фигурой (круг/квадрат/ромб) заданного цвета/прозрачности.
Опционально — уведомление в Telegram (только VIP/Admin).

Два независимых фильтра, каждый со своим вкл/выкл и своим визуалом:
- **Средний фильтр** (Medium): мин + макс объём.
- **Крупный фильтр** (Large): только мин объём.

---

## 2. Настройки фильтров

### Средний фильтр (Medium)
- enabled: вкл/выкл
- min_volume: число (обязательно)
- max_volume: число (обязательно)
- single_per_bar: чекбокс (одно выделение в баре)
- price_levels_merge: число объединяемых уровней (по умолчанию **3**)
- bid_ask_skew: % перевеса (по умолчанию **60**)
- delta_filter: опционально (фильтр по дельте)
- location: any | body | lower_wick | upper_wick (по умолчанию **any**)
- tg_notify: вкл/выкл (только VIP/Admin)

### Крупный фильтр (Large)
- enabled: вкл/выкл
- min_volume: число (обязательно; max НЕТ)
- single_per_bar: чекбокс
- price_levels_merge: число (по умолчанию **3**)
- bid_ask_skew: % (по умолчанию **60**)
- delta_filter: опционально
- location: any | body | lower_wick | upper_wick (по умолчанию **any**)
- tg_notify: вкл/выкл (только VIP/Admin)

### Визуал (отдельно для каждого фильтра)
- min_size, max_size (размер фигуры в px)
- shape: circle | square | diamond
- color_bid_imbalance (цвет при перевесе bid)
- color_ask_imbalance (цвет при перевесе ask)
- opacity (прозрачность)

---

## 3. Логика расчёта (по шагам)

Для каждого окна из `price_levels_merge` подряд идущих ценовых уровней в свече:

1. **Total volume** окна = Σ volume по уровням окна.
2. Проверка объёма:
   - Medium: `min_volume ≤ total ≤ max_volume`
   - Large: `total ≥ min_volume`
3. **Проверка перевеса bid/ask** (поле задаётся как ПОЛОЖИТЕЛЬНОЕ число %, напр. 60):
   ```
   bid_sum = Σ bid по окну
   ask_sum = Σ ask по окну
   bidImbalance = (bid_sum / total) * 100
   askImbalance = (ask_sum / total) * 100
   ```
   Условие выполнено, если `max(bidImbalance, askImbalance) > bid_ask_skew`.
   Доминирующая сторона = та, у которой % больше → выбирает цвет выделения.
4. **delta_filter** (если задан): дельта окна = ask_sum − bid_sum, проверить против условия.
5. **location**: окно засчитывается только если попадает в указанную зону свечи
   (any / тело / нижняя тень / верхняя тень). Тело = [min(open,close), max(open,close)],
   нижняя тень = [low, min(open,close)], верхняя тень = [max(open,close), high].
6. Если все проверки пройдены → пометить окно фигурой.
   - `single_per_bar = true` → в свече оставить только одно (с максимальным total).

---

## 4. Пример (из ТЗ)

Настройка Medium: min_volume = 180, price_levels_merge = 3, bid_ask_skew = 60.
Ситуация: на 3 подряд уровнях прошло bid = 150, ask = 50.
```
total = 200            → ≥ 180 ✔ (для max не выходит, если max позволяет)
bidImbalance = 150/200*100 = 75
askImbalance =  50/200*100 = 25
max(75,25) = 75 > 60   → перевес есть ✔, доминирует BID
```
→ выделить окно цветом `color_bid_imbalance`, фигурой согласно визуалу Medium.

---

## 5. Цвет выделения

- Доминирует bid → `color_bid_imbalance`
- Доминирует ask → `color_ask_imbalance`
Размер фигуры можно масштабировать от total в диапазоне [min_size, max_size]
(линейная интерполяция между min_volume и max_volume; для Large — от min_volume до visibleMax).

## 6. Telegram-уведомления (только VIP/Admin)

- Токен бота берётся из профиля пользователя (см. UserProfile).

## 7. Где считать

- **На бэкенде (Go)** для realtime + для уведомлений в TG (нужен серверный расчёт независимо от того, открыт ли у юзера график).
- На фронте движок только рисует результат (фигуры на Canvas), используя те же данные.
- Пресет по умолчанию кэшируется в Redis; права на изменение проверяет бэкенд (только VIP/Admin).
- Free/Pro используют пресет по умолчанию (Free — без права менять; Pro — может менять и сохранять свои пресеты, см. docs/04_TIERS.md).
