## 2. Архитектура и Логика Работы

PROCLUSTER спроектирован как **Full-Stack SPA-приложение** с возможностью переключения между автономным режимом (локальная симуляция) и внешним сетевым источником данных.

### Схема Клиент-Серверного Взаимодействия

1. **Серверная часть (`server.ts`):**
   - На этапе локальной разработки (`NODE_ENV !== "production"`) запускается в режиме интеграции с Vite (`createViteServer` в `middlewareMode`). Это позволяет "на лету" компилировать TypeScript на бэкенде и фронтенде.
   - В Production-окружении сервер компилируется в один автономный CJS-файл (`dist/server.cjs`), раздает оптимизированные статические файлы из директории `dist/` и обрабатывает API-эндпоинты.
   - Сервер содержит эндпоинты, симулирующие сбор тиков или парсинг исторических CSV файлов, а также формирует готовые структуры `ClusterCandle` на эндпоинте `/api/cluster-candles`.

2. **Запросы к Данным (`src/lib/api.ts`):**
   - Все основные запросы исторических кластеров осуществляются через функцию `getClusterCandles`.
   - В качестве базового адреса используется переменная окружения `VITE_API_BASE`. С её помощью происходит запрос к бэкенду.
   - Если переменная пуста, запрос идет к текущему хосту платформы по относительному URL: `/api/cluster-candles?symbol=...&interval=...`.

3. **Локальное Хранилище настроек (`src/lib/storage.ts`):**
   - Все пользовательские настройки (выбранные индикаторы, списки наблюдения, локальная история, лимиты, цветовые палитры и даже кастомные настройки индикаторов) сериализуются в JSON-строки и автоматически сохраняются в `localStorage`. Это защищает сессию пользователя при перезагрузке страницы.

## 3. Математика Данных и Алгоритмы Расчетов

Кластерный (или футпринт) анализ строится на распределении объема покупок и продаж внутри каждого шага цены свечи. В системе используются следующие базовые понятия:

### 1. Ячейка Кластера (`ClusterCell`)
Каждая свеча разбивается по горизонтальным ценовым уровням (ценовым шагам):
```typescript
export interface ClusterCell {
  price: number;            // Шаг цены (например, 68420.0, 68422.5 и т.д.)
  bid: number;              // Объем рыночных продаж (продажи "по рынку" в бид)
  ask: number;              // Объем рыночных покупок (покупки "по рынку" в аск)
  volume: number;           // Суммарный объем торгов на данном уровне (bid + ask)
  isPoc: boolean;           // Флаг максимального уровня объема в свече
  isBuyImbalance: boolean;  // Индикатор дисбаланса покупателей (Ask > Bid * Ratio)
  isSellImbalance: boolean; // Индикатор дисбаланса продавцов (Bid > Ask * Ratio)
}
```

### 2. Свеча Кластера (`ClusterCandle`)
Содержит в себе как классические OHLCV-метрики, так и специфические кластерные агрегации:
- **Volume:** $\sum \text{Cell.volume}$
- **Delta:** Сумма разностей покупок и продаж:
  $$\text{Delta} = \sum (\text{Cell.ask} - \text{Cell.bid})$$
- **POC Price (Point of Control):** Уровень цены, на котором зафиксирован максимальный горизонтальный объем внутри свечи.
- **Value Area (VAH, VAL):** Диапазон цен вокруг POC, вмещающий в себя 70% от совокупного объема свечи. Рассчитывается путем сортировки ячеек по убыванию объема и накопления суммы до тех пор, пока значение не превысит лимит в $70\%$.

## 4. Индикаторы на графике

### Индикатор: CVD (Cumulative Volume Delta)
CVD рассчитывается как накопительная сумма дельт всех свечей с момента начала сессии за весь доступный диапазон графика:
$$\text{CVD}_t = \text{CVD}_{t-1} + \text{Delta}_t$$
Отрисовывается в отдельной нижней панели графиком в виде линии или свечей дельты.

---

### Индикатор: **Volume on Chart (Объемы на графике)**
*Обновленный индикатор с расширенными параметрами кастомизации.*

В отличие от классического индикатора объема (который выводится в отдельном нижнем подвале), **Volume on Chart** накладывается непосредственно поверх зоны основного графика свечей, полупрозрачно заполняя нижнюю часть экрана:

1. **Одноцветный режим (Серый цвет по умолчанию):**
   Колонки гистограммы рендерятся в благородном сером цвете (адаптированном под темную и светлую темы), благодаря чему они не перегружают визуальное восприятие основных баров со спредом кластеров.
   
2. **Алгоритм Динамической Подсветки Дельты (Delta Threshold):**
   Позволяет выделять аномальные дисбалансы покупателей или продавцов. Если в рамках текущей свечи абсолютная дельта превышает задаваемый пользователем порог `volumeOnChartDeltaThreshold` (например, $500\text{ ETH}$ или $500\text{ BTC}$), цвет заливки колонки под свечой автоматически окрашивается:
   - **Зеленый цвет (Покупки преобладают):** если $\text{Delta} > \text{Threshold}$
   - **Красный цвет (Продажи преобладают):** если $\text{Delta} < -\text{Threshold}$
   - **Серый цвет (Стандартный):** если $|\text{Delta}| \le \text{Threshold}$

3. **Настройка Максимальной Высоты (`volumeOnChartMaxHeightPercent`):**
   Пользователь может задать процентную высоту от общей высоты основного графика (по умолчанию $20\%$), которую гистограмма объема не должна превышать.
   **Масштабирование по высоте:**
   - Система находит максимальный объем среди всех *видимых на экране* свечей: $\text{MaxVisibleVolume}$.
   - Коэффициент масштаба для каждого бара рассчитывается как:
     $$\text{BarHeight} = \left( \frac{\text{Candle.Volume}}{\text{MaxVisibleVolume}} \right) \times \left( \text{ChartHeight} \times \frac{\text{MaxHeightPercent}}{100} \right)$$
   Это предотвращает искажение масштаба, подстраиваясь под горизонтальный зум при деформировании осей.

---

### Индикатор: Cluster Search (Поиск Крупных Объемов)
Осуществляет поиск кластерных ячеек с экстремальной активностью по заданным фильтрам (например, фильтр среднего объема, минимальной дельты или минимального количества лотов в сделке). Найденные ячейки помечаются на Canvas специальными фигурами (кругами, квадратами или ромбами) разной прозрачности и цвета.

## 5. Графический Движок `ClusterChart.tsx` (Canvas Pipeline)

Отрезок графика отрисовывается на высокопроизводительном элементе `<canvas>` методом двумерного контекста (`2D Context`). Это гарантирует высокую кадровую частоту (FPS) даже при отображении тысяч горизонтальных уровней одновременно.

### Жизненный Цикл Отрисовки Кадра:

1. **Очистка Холста:** `ctx.clearRect(0, 0, width, height)`
2. **Расчет Масштаба (Scaling & Constraints):**
   - Вычисляется интервал видимости свечей (`startIdx` до `endIdx`) на основе текущего смещения прокрутки (`scrollOffset`) и ширины баров (`candleWidth`).
   - Находится локальный минимум и максимум цены на видимом участке для центрирования графика по вертикали.
3. **Отрисовка Опорной Сетки (Grid lines):** горизонтальные ценовые уровни и вертикальные временные шкалы.
4. **Рендеринг Индикатора Volume on Chart:** 
   Сначала вычисляется `visibleMaxCandleVolume` (чтобы избежать "перескока" масштаба при прокрутке на пустые зоны), а затем рисуются серые, зеленые или красные полупрозрачные прямоугольники колонок. Они рисуются **под** кластерами свечей.
5. **Рендеринг Свечей и Горизонтальных Уровней (Footprints):**
   - Если масштаб плотный (`candleWidth < 35`), свечи рисуются классическими барами или японскими свечами.
   - Если масштаб широкий (`candleWidth >= 35`), внутри каждой свечи отрисовываются горизонтальные уровни цен. Если шаг позволяет, выводятся числовые значения объема покупок/продаж внутри каждого кубика (например `5.2K / 1.1K`), а POC подсвечивается яркой контрастной рамкой.
6. **Оверлеи Индикаторов:** отрисовка слоев (Stacked Imbalances, Absorption, Reversal Clusters, Cluster Search).
7. **Отрисовка Оси Ординат (Price Axis, Справа)** и **Оси Абсцисс (Time Axis, Снизу)**.
8. **Элементы Интерактива:** Перекрестие курсора, линейка для измерения ценовых диапазонов, всплывающие подсказки.

## 6. Локальный Запуск и Развертывание

Для корректной сборки, запуска процесса статического анализа типов (linter) и компиляции приложения используются встроенные Node.js команды:

* **Установка всех зависимостей:**  
  `npm install`
* **Запуск сервера в режиме разработки (Dev Mode):**  
  `npm run dev` (применяет компилятор `tsx` для `server.ts` и Vite для реактивного обновления интерфейса)
* **Запуск проверки типов (Linter & Typecheck):**  
  `npm run lint`  *(Выполняет команду `tsc --noEmit`)*
* **Сборка проекта для Production (Production Build):**  
  `npm run build` *(Сначала собирает фронтенд-файлы в папку `dist/` с помощью `vite build`, затем упаковывает Express-сервер в единый файл `dist/server.cjs` за счет `esbuild` с внешними модулями)*
* **Запуск Скомпилированного Приложения:**  
  `npm run start` *(Запускает Node сервер из собранного файла `dist/server.cjs` на порту 3000)*
